//  結局（何に始まって何に終わったか）・・？

  //  何をトップダウンで想起し、ボトムアップで何(それ)を叶えたか。

折角来てくれたお客様を、御縁頂いた方々を、手ぶらで帰さない・・！
折角渡す名刺を、決して「燃えるゴミ」になんかさせない・・！
この２つが重なって、今回の「名刺」を「開発」するに至った。


//　どういうことか？

  //  名刺を渡される相手様 (ケースバイケースで立てる仮説)

名刺交換に際し、一度自分の名刺を受け取る相手様の立場になって考えてみた。
今回の相手様は遠路はるばる、時間、手数、足労をかけてわざわざ我々学生に会いに来て下さる企業様。
折角行って帰って来るなら、面白い学生の一人や二人と出会い、名刺を交換し、
覚えて帰りたいに決まっている。（それをする気がないなら抑々来ない。）
少なくとも、何かを得て、持ち帰りたいはずである。
少なくとも自分がその立場なら、そう考えたに違いない。

「自分が、企業交流会などで企業様に何か一つでもお渡しできるもの、
持って帰って頂けるものがあるとすれば、それは何か？」
その答えが、今回の名刺だった。

  // 「相手にどう価値を提供するか？」

    //  そもそも名刺の役割、名刺を使う・渡す目的（仮説）

結論として、名刺は以下2点を満たすべきだと考えた。

A　相手に、「燃えるゴミ」だと思われないこと。（捨てられにくい名刺であること）

B  相手に、なるべく強く・長い間、自分の名前、自分がどんな人間だったかや存在を憶えて帰ってもらうこと。

→　この名刺をくれた人、面白かったし、役に立った。価値を提供できる人だ。
　 そう思ってもらえる、御理解頂けることが一番それ(AとB)を叶えることが出来ると考えた。


    // QRコードの先のコンテンツ  (披露（提供）するコンテンツの意図・主眼)

単なる自慢、自分の力量、ポートフォリオ(作品集、実績)のひけらかしではなく、
相手が確実に価値を受け取れるようにする(受け取りやすくする)ことに主眼を置いた。
（形だけ真似ても、本質を押さえていなければ意味がない。）

つまり、相手が僕の名刺を受け取ったところから、
名刺の表、名刺の裏、QRコード、GitHubのReadMe、メインコンテンツへと進み、
最後再び、ReadMeの8番(最後)に至るまでを一本の線でつなぎ、
来て頂いた方々に、確実にこの一本の線を最初から最後まで、
脱落することなく歩んで頂いて、「楽しかった。面白かった。役に立った。」と、
満足してもらうこと。あくまでもそこに重きを置き、集中(コミット)した。


// 実際にこの名刺を受け取った、企業様の声

現に38歳業界未経験である僕のこの名刺に対し、
8名の方から反響を頂き、全員に好評だった。

企業様からは、チーム開発の重要性についての言及や、私の経歴・考え方への関心を寄せて頂く声が複数。
対面で話したAI活用にも関心を持って頂く等、様々な反響があった。

// 最後に

「想い・悲願・誓い」こそが「開発」の駆動力、起点、ルーツだということを、
今更ながらに気づかされるきっかけとなった。そしてそれこそが、AIによる代替を許さない、
人間が存在、介在する理由・価値ではないか。今回の体験を通して、そんな仮説を得た。
この仮説の検証を、今後も続けていく所存である。
